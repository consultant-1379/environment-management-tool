Overview

This page describes the permissions in the Environment Management Tool.

It describes what they allow each user to do and how to request them.

Purpose

The purpose of having roles and permissions is to improve the security in the tool, preventing malicious attacks.

It allows Bumblebee to lock down certain areas and to declutter EMT of information not needed by the user.

List of Permissions

The following are the current supported permissions in EMT:

  • OPS
    • Gives access to the environments page and API docs
    • This role is to simply distinguish between a guest and a non-guest (operations person) in EMT
    • Someone that doesn't have the OPS role will be treated as guest, meaning they can only view the Sessions page
  • MTE
    • Adds MTE environments to the default user view in EMT
    • Requires OPS role
