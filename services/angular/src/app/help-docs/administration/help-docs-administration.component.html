Overview

This page describes the administration features in EMT.

It describes the purpose of these features and how they work.

Purpose

The purpose of having an administration section in EMT is to provide the ability to:

  • Add Environments
  • Edit Environments
  • Remove Environments
  • Add User Roles
  • Remove User Roles

Add Environment

Steps

1 Enter environment name in the search box

  • Please note that this is case sensitive

2 Click on Add Environment button

  • If the Environment exists dialog box will be populated stating "Environment Already Exists in EMT"
  • If Environment does not exist, Add Environment form will be populated

3 Please enter the environment's properties

  • Property Description
    Name The name of an Environment.
    Test Phase The test phase associated to an Environment. The test phase is set when adding an Environment to EMT.
    Environment Type The type of Environment. The supported Environment types are: 1. Functional 2. Non-Functional 3. Functional/Non-Functional
    Platform Type The type of Platform. The supported Platform types are: 1. physical 2. vENM 3. SIENM 4. cENM

4 Click on Add Environment

Edit Environment

Steps

1 Enter environment name in the search box

  • Please note that this is case sensitive

2 Click the 'Edit Environment' button

  • If the Environment exists, a dialog box will be populated with the Environments current attributes.
  • If the Environment does not exist, the user will be notified by a pop up window.

3 Edit the environment's properties

  • Property Description
    Name The name of an Environment.
    Test Phase The test phase associated to an Environment. The test phase is set when adding an Environment to EMT.
    Product Set The product set deployed on the Environment.
    NOTE: Product set must be entered in the format: XX.XX.XXX eg. 19.10.10 / 19.10.101
    Environment Type The type of Environment. The supported Environment types are: 1. Functional 2. Non-Functional 3. Functional/Non-Functional
    Platform Type The type of Platform. The supported Platform types are: 1. physical 2. vENM 3. SIENM 4. cENM

  • If the environment is in busy state or has a session ongoing, the core information fields will be disabled and a tooltip will explain why.

4 Click on 'Edit Environment' to modify the Environment.

Remove Environment

Steps

1 Enter environment name in the search box

  • Please note that this is case sensitive

2 Click on Remove Environment button

  • If the environment does not exist, a dialog box will pop up stating that environment does not exist in EMT
  • If the environment is in BUSY state or it has any active session, a dialog box will pop up stating that the environment to remove can't be removed
  • If the environment exists, the Remove Environment form will be opened

3 Re-enter the environment's name

  • To confirm, please re-enter the name of the environment to be removed
  • The submit button will only be enabled in the name of the environment has been re-entered

4 Click on submit button to remove the Environment.

Update User Roles

The steps below outline how to update the roles assigned to a user.

By changing the roles, you update the environments that the user sees by default when they load EMT.

IMPORTANT:

  • If you do not have the sufficient permissions to update your own roles, that may mean the Admin tab in EMT may not be viewable to you. In this case please contact someone who has the sufficient permissions to view and make Admin changes.
  • By removing all roles you are telling EMT that the user should NOT have access to the administration page. Conversely, if you add one or more roles, you are telling EMT that the user should have access to the administration page.

Steps

1 Go to the Administration page

  • The link to the Administration page is present in the navigation bar
  • Click "User Roles"

2 Enter signum of person to edit roles for

  • In the input of the user roles administration page, you should input the signum to change user roles for
  • Signum should be 7 characters long
  • Signum should match signum present in LDAP
  • To search: press ENTER or click the search icon

3 Wait while the information is fetched

  • Searching data from LDAP can take a few seconds so please wait while the information loads

4 Select desired roles

  • Use the checkboxes to add or remove roles
  • Only the role checkboxes that are ticked will be assigned to the user
  • To select an entire group (e.g. MTV) please click the name of desired group (e.g. Maintrack Validation)
  • To select all roles click "Select All"
  • To revert to the original roles click "Reset"

5 Update roles

  • To update roles press "Update Roles"
  • Updating data can take some time so please wait
  • If the update is successful a success message will come up
